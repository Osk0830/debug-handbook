# Beginner Exercises

## 使い方

各Lessonを読んだ後、回答を見ずに取り組んでください。操作問題は任意のGit管理プロジェクトで実行します。

## 問題1

VS CodeとCLIの得意分野を、それぞれ2つずつ挙げてください。

## 問題2

単一ファイルだけではなく、プロジェクトルートを開くべき理由を3つ説明してください。

## 問題3

User設定とWorkspace設定へ、次の項目を振り分けてください。

- 個人のフォントサイズ
- チーム共通のPrettier指定
- 個人のテーマ
- 案件固有の検索除外

## 問題4

Problems、Output、Terminalの用途の違いを説明してください。

## 問題5

Command Paletteから次を実行してください。

1. User Settings(JSON)を開く
2. Keyboard Shortcutsを開く
3. 新しいTerminalを開く
4. WindowをReloadする

## 問題6

任意のプロジェクトで次のファイルをExplorerから見つけ、役割を書いてください。

- README
- 依存管理ファイル
- 起動設定
- エントリーポイント
- `.gitignore`

## 問題7

`src`配下のTypeScriptファイルだけから`TODO`を検索するIncludeパターンを書いてください。

## 問題8

`node_modules`、`vendor`、`dist`をSearchから除外するパターンを書いてください。

## 問題9

次の文字列をまとめて検索する正規表現を書いてください。

```text
console.log(
console.warn(
console.error(
```

## 問題10

`http://`と`https://`の両方へ一致する最小パターンを書いてください。

## 問題11

`TODO`、`FIXME`、`HACK`のいずれかへ単語単位で一致する正規表現を書いてください。

## 問題12

Searchと`rg`のどちらを使うか選び、理由を書いてください。

1. 一致箇所の前後コードを順番に読む
2. ファイルごとの一致件数を集計する
3. 置換Previewを確認する
4. 検索結果を別コマンドへ渡す

## 問題13

一括置換前後に実施すべき確認を、順番に6つ以上書いてください。

## 問題14

正規表現ではなく、Rename Symbolなどの言語機能を使うべき例を2つ挙げてください。

## 問題15 実践

以下の調査を行い、結果を記録してください。

```text
現象: 画面に表示されるエラーメッセージの発生条件を知りたい
```

記録項目:

```text
最初の検索語:
Include:
Exclude:
最初に見つけたファイル:
次の検索語:
呼び出し元:
調査結果:
```
