# デバッグの基本原則

## 推測で直さない

悪い例:

```text
たぶんこのif文が怪しい
↓
修正
↓
直らない
↓
別の場所を修正
↓
さらに壊れる
```

良い例:

```text
再現
↓
HTTP確認
↓
ログ確認
↓
コード位置特定
↓
SQL確認
↓
変数確認
↓
修正
↓
再確認
```

---

# 実務デバッグの全体フロー

```text
1. 現象を確認する
2. 再現条件を固定する
3. HTTP通信を確認する
4. curlで再現する
5. Dockerログを見る
6. rgで処理位置を探す
7. SQLでデータを確認する
8. Xdebugで変数を追う
9. Gitで履歴を確認する
10. 最小修正する
11. 再確認する
12. メモを残す
```

---

# 1. 現象を確認する

まず、何が起きているかを明確にします。

悪い表現:

```text
なんか動かない
```

良い表現:

```text
問い合わせフォームで送信ボタンを押すと、confirm.php へのPOST後に500エラーになる
```

確認する項目:

- URL
- 操作手順
- 入力値
- 発生タイミング
- 期待結果
- 実際の結果
- 再現頻度

---

# 2. 再現条件を固定する

毎回同じ条件で再現できるようにします。

例:

```text
URL: http://localhost:8081/wedding/inquiry/
入力:
  name=Test
  email=test@example.com
操作:
  確認画面へ進む
結果:
  500エラー
```

---

# 3. HTTP通信を確認する

Chrome DevToolsのNetworkタブで確認します。

見る項目:

- Method
- URL
- Status
- Request Headers
- Response Headers
- Payload
- Response
- Cookie

---

# 4. curlで再現する

ブラウザ操作をCLIで再現します。

```bash
curl -i -X POST "http://localhost:8081/wedding/inquiry/confirm.php" \
  -H "Content-Type: application/x-www-form-urlencoded" \
  --data "name=Test&email=test@example.com"
```

確認すること:

- HTTPステータス
- Location
- HTML本文
- エラー本文
- Cookie

---

# 5. Dockerログを見る

```bash
docker compose logs --tail=200 web-php8
docker compose logs -f web-php8
```

再現しながらログを見ます。

```bash
docker compose logs -f --tail=100 web-php8
```

---

# 6. rgで処理位置を探す

```bash
rg -n "confirm.php|execute.php|finish.php" inquiry data
rg -n "mail|send|subject" inquiry data
rg -n "INSERT|UPDATE|PDO|mysqli" inquiry data app app2
```

---

# 7. SQLでデータを確認する

DBに入ります。

```bash
docker compose exec db mysql -uroot -p
```

最新データ確認:

```sql
SELECT *
FROM inquiry
ORDER BY id DESC
LIMIT 10;
```

保存されているか、値が正しいかを確認します。

---

# 8. Xdebugで変数を追う

止める候補:

- 入力値を受けるところ
- バリデーション
- SQL実行直前
- SQL実行直後
- メール送信直前
- リダイレクト直前

見るもの:

- `$_POST`
- `$_SESSION`
- SQL文字列
- SQLパラメータ
- 戻り値
- 例外

---

# 9. Gitで履歴を確認する

```bash
git status
git diff
git log --oneline --graph --decorate -n 30
git blame path/to/file.php
```

いつから壊れたか分かる場合:

```bash
git bisect start
git bisect bad
git bisect good <正常だったコミット>
```

---

# 10. 最小修正する

修正は最小限にします。

悪い例:

```text
ついでに周辺処理も全部書き換える
```

良い例:

```text
原因箇所だけ直す
```

---

# 11. 再確認する

確認すること:

- 同じ手順で直っているか
- 別パターンでも壊れていないか
- ログにエラーがないか
- DBが正しいか
- メールが届くか
- Git diffが意図通りか

---

# 12. メモを残す

最低限これを残します。

```markdown
## 現象

## 再現手順

## 原因

## 証拠

## 修正内容

## 確認結果
```

# ケーススタディ1: 問い合わせフォームが500エラー

## 現象

問い合わせフォームでPOST後に500エラー。

## 調査

HTTP確認:

```bash
curl -i -X POST "http://localhost:8081/wedding/inquiry/confirm.php" \
  -H "Content-Type: application/x-www-form-urlencoded" \
  --data "name=Test&email=test@example.com"
```

ログ:

```bash
docker compose logs -f --tail=100 web-php8
```

コード検索:

```bash
rg -n "confirm.php|execute.php|PDO|INSERT|mail" inquiry data
```

SQL:

```sql
SELECT *
FROM inquiry
ORDER BY id DESC
LIMIT 10;
```

Xdebug:

- confirm.php
- execute.php
- INSERT直前
- mail送信直前

## ゴール

- 500の原因をログで説明できる
- どの行で失敗したか説明できる
- 修正後にcurlで200または302を確認できる

---

# ケーススタディ2: メールだけ送信されない

## 確認順

1. POSTは成功しているか
2. DB保存されているか
3. メール処理まで到達しているか
4. MailHogに届いているか
5. SMTP設定は正しいか

コマンド:

```bash
rg -n "mail|send|subject|SMTP|PHPMailer" .
docker compose logs -f mailhog
```

MailHog:

```text
http://localhost:8025
```

---

# ケーススタディ3: DBには保存されるが画面に表示されない

## 切り分け

- 保存処理は正常
- 取得処理が異常
- 表示条件が異常
- キャッシュが異常

SQL:

```sql
SELECT *
FROM target_table
ORDER BY id DESC
LIMIT 10;
```

コード検索:

```bash
rg -n "SELECT|WHERE|ORDER BY|target_table" .
```

---

# ケーススタディ4: APIだけ404

## 確認順

1. URL
2. Method
3. ルーティング
4. rewrite設定
5. nginx/apache設定
6. コンテナのポート

curl:

```bash
curl -i "http://localhost:8081/api/example"
```

検索:

```bash
rg -n "api/example|routes|rewrite|location" .
```

---

# ケーススタディ5: 302リダイレクトループ

## 確認順

1. Locationヘッダー
2. Cookie
3. Session
4. 認証判定
5. HTTPS判定

curl:

```bash
curl -I "http://localhost:8081/wedding/"
curl -L -I "http://localhost:8081/wedding/"
```

---

# ケーススタディ6: Xdebugが止まらない

確認:

```bash
docker compose exec web-php8 php -m | rg -i xdebug
docker compose exec web-php8 php -i | rg -n "xdebug.mode|xdebug.start_with_request|xdebug.client_host|xdebug.client_port"
docker compose exec web-php8 tail -100 /tmp/xdebug.log
```

確認ポイント:

- VS Code側が待ち受けているか
- portが一致しているか
- pathMappingsが正しいか
- リクエストが対象PHPを通っているか
