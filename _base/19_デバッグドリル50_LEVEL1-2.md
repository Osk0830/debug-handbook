
# 第19章 デバッグドリル50（LEVEL1〜2）

> **目的**: 現象から調査手順を組み立てる練習を行い、「どのコマンドを、なぜ打つか」を考える力を鍛える。

---

# 使い方

この章は**すぐに解説を見ない**でください。

1. 現象を読む
2. 仮説を3つ以上書く
3. 調査手順を書く
4. 使用するコマンドを書く
5. 原因を予想する
6. 解説を見る

# LEVEL1

## Drill01

### 現象

問い合わせフォーム送信後に500エラーになる。

### 自分で考える

- 最初に何を見る？
- curlで再現する？
- SQLは確認する？

---

### 解説

優先順位

1. HTTPステータス
2. Dockerログ
3. PHPログ
4. rgで処理検索
5. Xdebug

コマンド例

```bash
curl -i -X POST http://localhost:8081/wedding/inquiry/confirm.php
docker compose logs -f web-php8
rg -n "confirm.php|execute.php|INSERT|mail" .
```
