
# 第18章 実務ケーススタディ集

> **目的**: 実際の障害を題材に、「現象 → 仮説 → 調査 → 原因 → 修正 → 再確認」の流れを身につける。

---

# ケーススタディの進め方

各ケースで必ず次の順番で進めます。

1. 現象を整理
2. 再現
3. 仮説を立てる
4. 調査
5. 原因特定
6. 修正
7. リグレッション確認
8. 作業メモ

---

# Case1 問い合わせフォームが500エラー

## 現象

送信ボタン押下後に500エラー。

## 仮説

- PHPエラー
- SQLエラー
- メール送信
- 必須パラメータ不足

## 調査

```bash
curl -i -X POST http://localhost:8081/wedding/inquiry/confirm.php
docker compose logs -f web-php8
rg -n "confirm.php|execute.php|INSERT|mail" .
```

```sql
SELECT *
FROM inquiry
ORDER BY id DESC
LIMIT 10;
```

## 原因例

PDO例外でSQL実行失敗。

**仮説 → 検証 → 否定 → 次の仮説**

を素早く回すことが、一番重要なスキルです。
