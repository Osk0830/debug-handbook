# Lesson07 Git・デプロイ

## このLessonの目的

実務で起きやすい障害を、推測ではなく証拠に基づいて切り分けます。

## 基本フロー

1. 現象と期待値を言語化する
2. 再現条件を固定する
3. 観測ポイントを決める
4. 仮説を一つずつ検証する
5. 最小修正を行う
6. 正常系・異常系・境界値を再確認する

## ノック

### Knock 61: 修正したはずのコードが本番にない

**調査方針:** デプロイ対象コミット、ブランチ、成果物、サーバー上HEADを確認する。

**記録すること**

- 再現条件
- 確認した証拠
- 棄却した仮説
- 原因
- 修正内容
- 再発防止
