# Module25 実践デバッグ100本ノック

> 現象を見てすぐ修正するのではなく、再現・観測・仮説・検証・修正・再確認を100ケースで反復する実践Moduleです。

## 学習目標

- 不具合を再現条件へ分解できる
- 正常系と異常系を比較できる
- HTTP、PHP、SQL、JavaScript、Docker、Git、ログを横断して調査できる
- 原因と症状を区別できる
- 最小修正と再発確認を行える
- 調査結果を第三者へ説明できる

## 進め方

各ノックでは、最初に答えを見ず、必ず調査メモを残してください。

```text
事実:
仮説:
確認方法:
結果:
次の行動:
```

100本を一度で終える必要はありません。1日3〜5本ずつ、実際にコマンドやDevToolsを操作しながら進めます。
