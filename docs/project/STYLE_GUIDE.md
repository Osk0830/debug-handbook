# STYLE GUIDE

## 文章トーン

- 日本語
- です・ます調
- 実務寄り
- 読みやすく短めに書く
- 難しい用語は避けずに説明する
- 「なぜ使うのか」を必ず書く

## 見出しルール

- `#` はページタイトルのみ
- `##` は主要セクション
- `###` は補足セクション
- 見出しは短くする

## コードブロック

コマンドは言語指定を付けます。

```bash
pwd
ls -lah
```

SQL:

```sql
SELECT *
FROM users
LIMIT 10;
```

JavaScript:

```js
const res = await fetch('/api/users');
```

PHP:

```php
var_dump($user);
```

## 危険な操作

危険な操作には必ず注意を書く。

例:

> **注意**
> `rm -rf` は実行前に必ず `pwd` と `ls -lah` を確認します。

## 実務メモ

実務的な補足は以下の形式にします。

> **実務メモ**
> 現場では、修正より先に証拠を集めます。

## よくある失敗

各Lessonには可能な限り「よくある失敗」を入れます。

例:

```markdown
## よくある失敗

- エラーログを見ずにコードだけ読む
- 画面表示だけで判断する
- 修正後に再確認しない
```

## ハンズオン

ハンズオンには必ず「結果から何が分かるか」を書かせます。

```markdown
## 実行したこと

## 結果

## 分かったこと

## 次に確認すること
```

## コミットメッセージ

推奨:

```text
Add Module01 CLI
Expand Module05 curl examples
Fix MkDocs config
Add WordPress case study
```

## リンク方針

- 公式ドキュメントを優先する
- 補助として良質な記事を追加する
- 古くなりやすい記事はResourcesにまとめる
