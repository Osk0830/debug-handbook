# ログ解析 FAQ

## `tail -f`と`tail -F`の違いは何ですか

`-F`はログローテーションなどでファイルが置き換わった場合にも再追跡しやすい指定です。

## Access Logに500があります。原因は分かりますか

Access Logだけでは通常、失敗した事実までです。同時刻・Request IDのError LogやApplication Logを確認します。

## ログに何も出ません

別ファイル、stdout / stderr、別コンテナ、別SAPI、ログレベル、権限、ローテーション後のファイルを確認します。

## Dockerログはどこにありますか

まず`docker compose logs`または`docker logs`で確認します。ホスト側の保存場所を直接読むより、Dockerのインターフェースを使う方が安全です。

## 時刻が9時間ずれています

UTCとJSTの可能性があります。OS、コンテナ、PHP、DB、ログフォーマットのタイムゾーンを比較します。

## General Logを常時有効にしてよいですか

大量出力と性能影響があるため、原則として調査時間を限定します。

## ログをそのままチャットへ貼ってよいですか

Authorization、Cookie、Token、パスワード、個人情報を確認し、共有用コピーをマスキングしてください。

## grepの結果が多すぎます

日付、URL、ステータス、Request ID、例外名の順で段階的に絞ります。
