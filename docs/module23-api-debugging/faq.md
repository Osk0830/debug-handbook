# APIデバッグ FAQ

## curlでは成功するのにブラウザでは失敗します

CORS、CookieのSameSite、Mixed Content、ブラウザキャッシュ、Service Workerを確認します。

## 400と422の違いは何ですか

一般に400はリクエスト形式、422は形式は読めるが入力値が不正な場合に使われます。実際のAPI仕様を優先してください。

## 401と403の違いは何ですか

401は認証できていない状態、403は認証済みでも操作を許可されない状態です。

## `curl -k`を使えば直りますか

証明書検証を無効にするだけです。原因確認の一時用途に限定し、恒久対応にしないでください。

## リダイレクト後の結果だけ見たいです

`-L`を使えます。ただしデバッグでは、まず追従せず各レスポンスを確認してください。

## JSONなのにPHPの`$_POST`が空です

`application/json`のボディは通常`$_POST`へ入りません。`php://input`から読み取ります。

## トークンをログへ出してよいですか

原則として出さないでください。必要な場合もマスキング・ハッシュ化・短時間保持を徹底します。

## 502と504の違いは何ですか

502は上流への接続失敗や不正応答、504は上流応答待ちのタイムアウトが主な意味です。
