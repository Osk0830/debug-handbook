# Lesson01 VS Codeとは

## ゴール

VS Codeを単なるエディタではなく、実務デバッグの統合環境として説明できるようになります。

## VS Codeの役割

VS Codeは、次の作業を1つの画面に集約できます。

- コード閲覧
- コード検索
- CLI操作
- Git差分確認
- ログ確認
- Docker操作
- Debugger
- AI支援
- Problems確認

## 実務ではいつ使う？

- PHPフォームの500エラーを追う
- ReactのAPIレスポンスとStateを比較する
- WordPressのテンプレート階層を追う
- Dockerログとコードを同時に確認する
- Git履歴から「いつ壊れたか」を確認する

## 実務デバッグの流れ

```text
現象確認
  ↓
Search
  ↓
Terminal
  ↓
Logs
  ↓
Debugger
  ↓
Git
  ↓
修正
  ↓
再確認
```

## VS CodeとCLIの関係

VS CodeとCLIは競合しません。

|用途|VS Code|CLI|
|---|---|---|
|コードを読む|強い|弱い|
|検索結果から開く|強い|普通|
|大量検索|普通|`rg` が強い|
|結果の加工|弱い|強い|
|デバッグ|強い|用途次第|
|自動化|普通|強い|

## ハンズオン

1. VS Codeでプロジェクトを開く
2. Searchで任意の関数名を検索
3. 統合ターミナルで同じ文字列を `rg` 検索
4. 両者の違いをメモする

```bash
rg -n "function" .
```
