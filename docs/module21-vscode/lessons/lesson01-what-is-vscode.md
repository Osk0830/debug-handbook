# Lesson01 VS Codeとは

## ゴール

- VS Codeを「コードを書く場所」ではなく「調査の中心」と説明できる
- エディタ、IDE、CLIの違いを大まかに説明できる
- デバッグ時にVS CodeとTerminalを組み合わせられる

## VS Codeの位置づけ

Visual Studio Codeは、軽量なコードエディタを土台に、拡張機能によってIDEに近い機能を追加できる開発環境です。

標準状態でも次の機能を利用できます。

- ファイル編集
- ワークスペース全体検索
- Git差分の表示
- 統合Terminal
- ブレークポイントを使ったデバッグ
- Problemsによるエラー一覧
- 設定とキーバインドの管理

拡張機能を追加すると、PHP、JavaScript、TypeScript、Python、Dockerなどの開発支援を強化できます。

## デバッグの司令塔として使う

たとえばPHPアプリケーションで500エラーが発生したとします。

```text
ブラウザで500を確認
  ↓
SearchでURLや処理名を検索
  ↓
Terminalでコンテナログを確認
  ↓
関係する関数へブレークポイントを置く
  ↓
変数の値を確認
  ↓
修正後にGit差分を確認
```

この一連の作業を、VS Codeから大きく離れずに実行できます。

## VS CodeとCLIの使い分け

| 作業 | VS Code | CLI |
|---|---:|---:|
| ファイルを読みながら検索 | ◎ | ○ |
| 複数ファイルの差分確認 | ◎ | ○ |
| ログの絞り込み・加工 | △ | ◎ |
| 大量ファイルの高速検索 | ○ | ◎ |
| ブレークポイントデバッグ | ◎ | △ |
| 自動化・繰り返し処理 | △ | ◎ |
| 初学者が構造を把握する | ◎ | △ |

どちらか一方に統一する必要はありません。**見ながら調べる作業はVS Code、繰り返し・加工・自動化はCLI**を基本にすると効率的です。

## フォルダを開くことが重要

単一ファイルだけを開くと、検索、Git、設定、補完の対象範囲が限定されます。基本はプロジェクトルートを開きます。

```bash
cd ~/projects/sample-app
code .
```

`code .` の `.` は「現在のディレクトリ」を意味します。

## ワークスペースとは

VS Codeが現在扱っている作業範囲をワークスペースと呼びます。単一フォルダを開いた場合、そのフォルダ以下が検索・Git・設定の基準になります。

確認したい代表ファイルは次のとおりです。

```text
README.md
.git/
.gitignore
package.json
composer.json
docker-compose.yml / compose.yml
.env.example
src/ / app/
```

## ハンズオン

### 1. プロジェクトを開く

```bash
mkdir -p ~/debug-handbook-labs/vscode-lesson01
cd ~/debug-handbook-labs/vscode-lesson01
printf '# VS Code Lab\n' > README.md
git init
code .
```

### 2. VS CodeとCLIで同じ情報を確認する

VS CodeではExplorerから`README.md`を開きます。Terminalでは次を実行します。

```bash
pwd
ls -la
git status
```

### 3. 調査経路を記録する

次の問いに答えてください。

- ファイルの場所を知っている場合、ExplorerとSearchのどちらを使うか
- 文字列の場所を知らない場合、何を使うか
- Gitの未コミット差分をどこで確認するか

## よくある誤解

### VS Codeだけで開発を完結させるべき

CLIの方が速い処理もあります。VS CodeはCLIを置き換えるのではなく、CLIを統合して使いやすくする環境です。

### 拡張機能を大量に入れるほど便利

拡張機能は起動速度、補完、フォーマット、キーバインドへ影響します。役割が重複する拡張は避け、必要性を説明できるものだけ導入します。

## 確認問題

1. プロジェクトルートを開く利点を2つ挙げてください。
2. ログから特定文字列だけを抽出する場合、VS CodeとCLIのどちらが向いていますか。
3. VS Codeをデバッグの司令塔と呼べる理由を説明してください。

## まとめ

- VS Codeは編集・検索・Terminal・Git・Debuggerを統合する
- 単一ファイルではなくプロジェクトルートを開く
- VS CodeとCLIは競合する道具ではなく、相互補完する道具である
