# Lesson18 Extensionsを選定・管理する

## このLessonのゴール

- 拡張機能の権限と信頼性を評価できる
- 競合や負荷を切り分けられる
- プロジェクト推奨拡張を管理できる

## はじめに

拡張機能はVS CodeをIDEへ近づけますが、コード実行、ファイルアクセス、ネットワーク通信を伴うものもあります。「便利そうだから全部入れる」のではなく、目的、提供元、更新状況、権限、代替手段を確認します。

## 選定基準

提供元、MarketplaceのVerified表示、更新頻度、利用者数、Issue、ライセンス、必要権限を確認します。利用者数だけで安全性を断定しません。公式または言語コミュニティで標準的なものを優先します。

## 拡張の実行場所

ローカル、SSH、WSL、Dev Containerなど、拡張がどこで動くかによって挙動が変わります。Remote環境では「Install in ...」が必要な場合があります。

## 競合の切り分け

同じ言語で複数のフォーマッターやリンターを有効にすると競合しやすくなります。Extension Bisect、Disable All Installed Extensions、Empty Profileを使って原因を絞ります。

## パフォーマンス

Developer: Show Running Extensionsで起動時間や処理時間を確認できます。巨大ワークスペースではファイル監視、インデックス、Git解析が負荷要因になります。まず測定し、無効化前後を比較します。

## 推奨拡張

`.vscode/extensions.json` に `recommendations` と `unwantedRecommendations` を記載できます。導入を強制するものではないため、READMEに用途と必須度を補足します。

## 更新方針

自動更新は修正を受け取りやすい一方、業務中の挙動変化も起こります。重要案件ではバージョン、更新日、ロールバック方法を記録し、問題が出たらDisable Auto Updateや特定バージョン導入を検討します。

## ハンズオン

1. 現在有効な拡張を用途別に分類します。
2. 同じ役割の重複拡張がないか確認します。
3. Running Extensionsで負荷を確認します。
4. サンプルのextensions.jsonへ推奨拡張を2つ追加します。
5. 1つの拡張を無効化し、問題や速度が変わるか比較します。

## 確認問題

1. 拡張機能選定で確認する項目を3つ挙げてください。
2. Extension Bisectは何を調べる機能ですか。
3. 推奨拡張と必須依存ツールはどう違いますか。
