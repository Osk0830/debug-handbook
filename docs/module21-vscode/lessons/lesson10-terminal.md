# Lesson10 統合Terminalで検証を回す

## このLessonのゴール

- 統合Terminalを安全に操作できる
- プロジェクトの実行・ログ確認・検索をエディタと往復できる
- 複数Terminalとプロファイルを用途別に使い分けられる

## はじめに

VS Codeの統合Terminalを使うと、コード、検索結果、Git差分、コマンド出力を同じウィンドウで確認できます。重要なのは「Terminalがあること」ではなく、現在地と実行環境を常に確認し、再現可能なコマンドで検証することです。

## 基本操作

macOSは ``Ctrl + ` ``、Windows/Linuxも ``Ctrl + ` `` でTerminalを開閉します。新規Terminal、分割、名前変更、終了を使い分けます。Terminalを増やしすぎたら、用途が分かる名前へ変更します。

## 現在地と環境を確認する

最初に `pwd`、`git status`、必要に応じて `node -v`、`php -v`、`docker compose ps` を実行します。異なるリポジトリやホスト環境でコマンドを実行する事故を防ぎます。

## コマンド履歴を証拠にする

再現に使ったコマンドはREADME、Issue、作業メモへ残します。長いワンライナーより、段階ごとに実行して結果を確認します。秘密情報を含むコマンドは履歴やスクリーンショットへ残さないでください。

## 複数Terminalの運用

例として `app`、`test`、`log` の3つに分けます。1つ目で開発サーバー、2つ目でテスト、3つ目でログ監視を行うと、停止や再実行の対象を間違えにくくなります。

## エディタとの連携

ファイルパスはCmd/Ctrlを押しながら選択すると該当箇所へ移動できる場合があります。エラー出力のパス形式が認識されない場合はTaskのproblemMatcherを検討します。

## Terminalが起動しない場合

既定シェル、ログインシェル設定、PATH、shell integration、拡張機能を確認します。外部Terminalでは動くかを比較すると、VS Code固有かシェル全体の問題かを切り分けられます。

## ハンズオン

1. プロジェクトをVS Codeで開き、`pwd` と `git status` を確認します。
2. Terminalを3つ作り、`app`、`test`、`log` と名前を付けます。
3. `rg "TODO|FIXME" .` を実行して結果からファイルを開きます。
4. テストまたはlintを実行し、失敗箇所へ移動します。
5. 実行したコマンドと結果を短い調査メモへ残します。

## 確認問題

1. Terminalを開いた直後に確認すべき項目は何ですか。
2. 複数Terminalへ名前を付ける理由は何ですか。
3. 外部Terminalと結果を比較する目的は何ですか。
