# Lesson25 ケーススタディ: Reactの表示不具合を追う

## このLessonのゴール

- Reactのデータ取得・状態・描画を分けて調査できる
- Browser DevToolsとVS Codeを往復できる
- 非同期処理と条件分岐の不具合を再現できる

## はじめに

題材は「APIレスポンスには画像URLがあるのに、一部カードだけ画像が表示されない」です。Reactの問題に見えても、URL、HTTP応答、CORS、状態更新、条件分岐、CSSのどこでも起こり得ます。

## 現象を分類する

DOMにimgがない、imgはあるがsrcが空、404、CORS、画像は読み込まれたがCSSで見えない、という状態を分けます。スクリーンショットだけでなくElementsとNetworkを確認します。

## Networkを確認する

リクエストURL、ステータス、Content-Type、リダイレクト、キャッシュ、Initiatorを確認します。APIレスポンスに値があることと、ブラウザが画像取得に成功することは別です。

## コード検索

レスポンス型、変換関数、画像コンポーネント、フォールバック処理をシンボル検索します。`imageUrl`、`thumbnail`、`onError` など、値が名前変更される境界を追います。

## Debugger

ブラウザデバッグ構成やJavaScript Debug Terminalを使い、描画前のpropsとstateを確認します。source mapが正しく、元のTypeScript/JSXで停止するか確認します。

## React特有の注意

Strict Modeによる開発時の追加実行、非同期更新、古いclosure、依存配列、keyの不適切利用を確認します。Consoleログの回数だけで本番挙動を断定しません。

## 画像エラーの設計

`onError` で無限ループしないフォールバックを設定し、壊れたURLを監視できるログやメトリクスを検討します。alt属性やレイアウトシフトも確認します。

## ハンズオン

1. 不具合カードを1件固定し、正常カードと比較します。
2. Networkで画像リクエストを確認します。
3. APIレスポンスからimgのsrcまで値の変換経路を追います。
4. 描画直前にブレークポイントを置きます。
5. 404・空値・CSS非表示それぞれのテストを追加します。

## 確認問題

1. APIレスポンスにURLがあっても画像が表示されない理由を3つ挙げてください。
2. React Strict Modeの開発時挙動で注意する点は何ですか。
3. onErrorフォールバックで無限ループを避けるにはどうしますか。
