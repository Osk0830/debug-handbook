# Lesson22 Continueなど外部AI拡張を評価する

## このLessonのゴール

- 外部AI拡張の構成要素を理解できる
- モデル・接続先・データ送信範囲を評価できる
- Copilot等と役割を重複させず運用できる

## はじめに

Continueのような外部AI拡張は、利用モデルや接続先を柔軟に構成できる一方、設定と情報管理の責任も増えます。このLessonでは特定製品の導入を必須とせず、AI拡張を評価する観点を学びます。

## 評価する要素

拡張の提供元、モデルプロバイダー、認証方法、送信データ、ログ保存、テレメトリ、ローカルモデル対応、ライセンスを確認します。「ローカルモデル対応」と「すべてが端末内だけで完結」は同義ではありません。

## コンテキスト機能

ファイル、選択範囲、Git差分、Terminal出力、ドキュメントなどをコンテキストへ追加できる場合があります。便利な一括追加ほど送信範囲が広がるため、必要最小限にします。

## 設定ファイル

設定形式や保存場所はバージョンで変わる可能性があります。公式ドキュメントと導入中の拡張バージョンを確認し、古い設定例を盲目的にコピーしません。APIキーは秘密管理へ置きます。

## 複数AI拡張の競合

インライン補完、Chat UI、キーバインド、コードアクションが重複することがあります。用途を決め、不要な機能を無効化します。問題時は片方ずつ無効化して再現を比較します。

## 導入判断

柔軟性が必要か、管理コストを負担できるか、会社の承認済みプロバイダーか、監査ログが必要かを確認します。個人の利便性だけで業務環境へ導入しません。

## ハンズオン

1. 導入候補AI拡張について、提供元とデータ送信先を調べます。
2. 利用予定モデルと認証情報の保存方法を整理します。
3. 送信してよいデータ・禁止データを表にします。
4. 既存AI拡張との機能重複を確認します。
5. 導入・見送りの判断を5行以内で記録します。

## 確認問題

1. 外部AI拡張で確認すべき情報管理項目を挙げてください。
2. 複数AI拡張を同時利用すると何が競合しますか。
3. ローカルモデル利用時も確認が必要な点は何ですか。
