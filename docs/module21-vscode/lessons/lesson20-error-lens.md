# Lesson20 Error Lensで診断を見逃さない

## このLessonのゴール

- Error Lensの役割を説明できる
- 表示過多を抑える設定ができる
- Problemsパネルとの使い分けができる

## はじめに

Error Lensは診断メッセージをコード行の近くへ目立つ形で表示する拡張機能です。問題へ早く気づける一方、大量の既存警告があるプロジェクトでは情報過多になります。

## 役割

VS Codeや言語拡張が発行したDiagnosticsをインライン表示します。Error Lens自体がTypeScriptやPHPの解析を行うわけではありません。発生元のリンターや言語サーバーが正しく動いている必要があります。

## Problemsとの違い

Error Lensは現在見ているコード上で気づきやすく、Problemsはプロジェクト全体を一覧しやすいという違いがあります。局所修正と全体確認で使い分けます。

## 表示を調整する

エラーだけ表示、警告の装飾を弱める、特定ソースを除外するなどの調整が可能です。設定キーは拡張の現行ドキュメントとSettings UIで確認し、古い記事をそのままコピーしません。

## 既存警告が多い現場

変更対象ファイルへ範囲を絞り、baselineを把握します。全警告を一度に直す変更と機能修正を混ぜるとレビューが難しくなるため、別コミット・別Issueへ分けます。

## 無効化の判断

画面が読みづらい、スクリーンリーダー利用へ影響する、診断の重複が多い場合はWorkspace単位で無効化を検討します。便利さより調査精度と可読性を優先します。

## ハンズオン

1. Error Lensを有効にし、警告を含むファイルを開きます。
2. Problemsパネルと表示件数を比較します。
3. エラーだけが目立つ設定へ調整します。
4. 拡張を無効化し、作業速度と可読性を比較します。
5. チーム推奨にするか、個人任意にするか理由付きで判断します。

## 確認問題

1. Error Lensは診断を自分で生成しますか。
2. Problemsとの使い分けを説明してください。
3. 既存警告を機能修正と分ける理由は何ですか。
