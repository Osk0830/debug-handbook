# Lesson29 総合演習

## このLessonのゴール

- 検索から修正確認まで一連の調査を実践できる
- 複数ツールから得た証拠を統合できる
- 調査結果を第三者へ説明できる

## はじめに

このLessonでは、これまで学んだSearch、Problems、Outline、Terminal、Debugger、Git履歴、Taskを組み合わせます。正解コードだけでなく、調査順序と根拠を評価します。

## 演習A: 設定値が反映されない

環境変数 `API_BASE_URL` を変更しても古い接続先へアクセスします。検索対象、設定スコープ、開発サーバー再起動、ビルド時注入、キャッシュを調査し、原因候補を優先順位付きで整理してください。

## 演習B: 一部データだけ500

特定の顧客IDだけAPIが500になります。Problemsは0件です。ログ、条件付きブレークポイント、DBデータ、Git履歴を使い、正常IDとの差分を特定してください。

## 演習C: CIだけlint失敗

ローカルのVS Codeでは問題がありませんがCIでlintが失敗します。拡張機能、Task、package script、Nodeバージョン、設定ファイル、対象パスを比較してください。

## 演習D: VS Codeが重い

特定のモノレポだけ入力が遅延します。Running Extensions、Process Explorer、検索・監視除外、Git、Empty Profileを使って計測計画を作ってください。

## 提出形式

各演習について、現象、再現条件、事実、仮説、検証、原因、修正、再発防止を記載します。推測と確認済み事実を明確に分けてください。

## ハンズオン

1. 4演習から2つ選びます。
2. 最初の15分はコードを変更せず証拠を集めます。
3. 調査に使ったVS Code機能とCLIコマンドを記録します。
4. 最小修正と回帰確認を実施します。
5. 別の人が再現できる形式で報告書を作ります。

## 確認問題

1. Problemsが0件でも障害が存在する理由は何ですか。
2. ローカルとCIの差を比較するときの軸を挙げてください。
3. 調査報告で事実と仮説を分ける理由は何ですか。
