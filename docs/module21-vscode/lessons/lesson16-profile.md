# Lesson16 Profilesで環境を分離する

## このLessonのゴール

- Profileへ含まれる要素を説明できる
- 用途別Profileを作成できる
- Settings Syncとの関係を理解できる

## はじめに

Profilesを使うと、設定、拡張機能、キーボードショートカット、UI状態などを用途別に分けられます。PHP案件、React案件、教材執筆などで必要な拡張が大きく異なる場合に有効です。

## Profileの用途

拡張機能の競合を減らす、初学者向け最小環境を作る、デモ環境を再現する、といった用途があります。プロジェクトごとの細かな設定はWorkspace設定へ置き、Profileへ詰め込みすぎないようにします。

## 作成と切替

ManageメニューからProfilesを開き、新規作成します。空のProfile、現在のProfileからコピー、テンプレート利用を選べます。切替後はウィンドウ名や導入拡張を確認します。

## エクスポートと共有

Profileはエクスポートできます。共有前にアカウント固有情報、社内専用拡張、不要な設定が含まれないか確認します。Profileは依存ツール本体まで配布するものではありません。

## Settings Syncとの違い

Settings Syncは自分の複数端末間で設定を同期する仕組みです。Profileは1つのVS Code内で用途を分離する仕組みです。同期対象にProfileをどう含めるかを意識します。

## トラブル切り分け

拡張機能由来の問題が疑われるとき、Empty Profileで再現するか確認すると切り分けが速くなります。通常環境を壊さず、最小構成で試せます。

## ハンズオン

1. `Web Debug` というProfileを作成します。
2. 必要最小限の拡張だけを有効にします。
3. 通常Profileと設定・拡張数を比較します。
4. Empty Profileで既知の問題が再現するか確認します。
5. 共有可能なProfileとしてエクスポート前チェックを行います。

## 確認問題

1. ProfileとWorkspace設定はどう使い分けますか。
2. Empty Profileがトラブル調査に有効な理由は何ですか。
3. Profileを共有しても別途必要になるものを挙げてください。
