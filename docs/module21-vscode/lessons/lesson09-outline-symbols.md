# Lesson09 Outlineとシンボル検索でコードを俯瞰する

## このLessonのゴール

- 長いファイルの構造をOutlineで把握できる
- ファイル内・ワークスペース全体のシンボルへ移動できる
- 定義・参照・呼び出し階層を調査に使える

## はじめに

デバッグでは、ファイルを上から順番に読むより、関数、クラス、メソッド、変数などの「シンボル」を手掛かりに移動した方が速くなります。Outlineとシンボル検索は、コードの地図として機能します。

## Outlineビュー

Explorer内のOutlineには、現在のファイルに含まれるクラス、関数、メソッド、プロパティなどが表示されます。並び順は位置・名前・種類で切り替えられます。

シンボルが表示されない場合は、ファイルの言語モードと対応する言語拡張を確認します。

## ファイル内シンボル

macOSは `Cmd + Shift + O`、Windows/Linuxは `Ctrl + Shift + O` です。入力欄で名前を絞り込み、`:` を入力すると種類ごとにグループ化できます。長いPHPクラスやReactコンポーネントで特に有効です。

## ワークスペースシンボル

macOSは `Cmd + T`、Windows/Linuxは `Ctrl + T` です。プロジェクト全体からシンボル名を検索します。ファイル名や文字列検索より、「実装の定義」を探したい場合に適しています。

## 定義・実装・参照

`F12` で定義へ移動、`Option + F12` / `Alt + F12` でPeek Definition、`Shift + F12` で参照一覧を表示します。インターフェースや抽象化を使うコードでは、Go to Implementationsも使います。

## Call Hierarchy

関数のコンテキストメニューから呼び出し階層を表示すると、「誰がこの関数を呼ぶか」「この関数が何を呼ぶか」を追えます。障害の入口から副作用までを調べるときに有効です。

## ハンズオン

1. 100行以上のクラスまたはコンポーネントを開きます。
2. Outlineから目的のメソッドへ移動します。
3. ファイル内シンボル検索で同じ場所へ移動します。
4. 参照一覧とCall Hierarchyを開き、呼び出し元をメモします。
5. 文字列検索で得た結果と比較し、どちらが調査に適するか整理します。

## 確認問題

1. シンボル検索と通常の文字列検索はどう違いますか。
2. Peek Definitionを使う利点は何ですか。
3. Call Hierarchyが有効な障害例を挙げてください。
