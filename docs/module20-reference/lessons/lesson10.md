# Lesson10 Reference総復習

## ゴール

Reference総復習を、実務で使える形で整理します。

## このLessonで身につくこと

- このテーマの目的を説明できる
- 実務での使いどころを判断できる
- 最初に確認すべき証拠を選べる
- 調査結果をメモできる

## 実務ではいつ使う？

Module20 Reference は、実際の障害対応や復習で使います。

特に以下のような時に役立ちます。

- 何から確認すべきか迷ったとき
- 似た障害を過去に調べたことがあるとき
- コマンドや判断フローをすぐ見返したいとき
- 学習内容を実務に接続したいとき

## 基本

デバッグでは「正解を暗記する」のではなく、確認順序を持つことが重要です。

```text
現象
  ↓
証拠
  ↓
仮説
  ↓
検証
  ↓
修正
  ↓
再確認
```

## よくある失敗

- 問題文や現象を読み飛ばす
- 最初の仮説に固執する
- ログやHTTPを確認せずにコードを見る
- 結果をメモしない
- 修正後に同じ手順で再確認しない

## ハンズオン

以下のテンプレートを使って、1つ調査メモを書いてください。

```markdown
## 現象

## 仮説

1.
2.
3.

## 最初に確認する証拠

## 実行するコマンド・操作

## 結果

## 次に確認すること
```

## 演習

1. このLessonはどんな場面で使いますか？
2. 最初に確認する証拠は何ですか？
3. その証拠から何が判断できますか？
4. 次に確認すべきことは何ですか？

## 模範解答

現象によって変わりますが、基本は外側から内側へ確認します。

```text
Browser
  ↓
HTTP
  ↓
Web Server
  ↓
Application
  ↓
DB / Mail / External API
```

## FAQ

### Q. すぐに原因が分からない場合は？

仮説を増やし、1つずつ検証します。原因を当てるより、候補を減らすことが重要です。

## 学習ログ

```markdown
## 学習日

## 試したこと

## 分からなかったこと

## 実務で使えそうな場面
```
