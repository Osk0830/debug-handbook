# curlデバッグフロー

## ゴール

curlデバッグフローを実務デバッグで使えるレベルまで理解します。

## このLessonで身につくこと

- 基本的な役割を説明できる
- 実務で使う場面を判断できる
- 最低限のコマンドや確認項目を使える
- よくある失敗を避けられる

## 実務ではいつ使う？

Module05 curl完全攻略 は、障害調査の入口または切り分けで頻繁に使います。

特に以下のような場面で使います。

- 何が起きているか確認したいとき
- ブラウザ表示だけでは原因が分からないとき
- ログやレスポンスなどの証拠を集めたいとき
- 修正後に再現確認したいとき

## 基本

まずは「何を確認するためのものか」を意識します。

コマンドや用語は丸暗記ではなく、**どの証拠を得るために使うのか**を考えます。

## 実務メモ

デバッグでは、いきなり修正しません。

まず現象を確認し、次に証拠を集めます。

```text
現象
  ↓
証拠
  ↓
仮説
  ↓
検証
  ↓
修正
```

## よくある失敗

- 目的を決めずにコマンドを打つ
- エラーメッセージを読まない
- 画面だけ見て判断する
- 修正後の再確認をしない

3. 推測だけで修正することです。
